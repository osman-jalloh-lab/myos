Dashboards should be compact, scannable, and action oriented.

Use compact headers that leave room for actual work. Put key analytics, health,
status, or priority indicators above the fold. Show what changed recently with a
live activity feed or recent events list.

Queue and status panels should show counts, active items, failures, and next
actions. Chat can be present, but it should be secondary to the operational
surface unless the app is explicitly a chat product.

Avoid fake metrics. If data is demo data, make it plausible and make every
dashboard panel explain a real decision.
