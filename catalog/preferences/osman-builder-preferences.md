Osman prefers Builder output that feels complete enough to use immediately.

Build the actual app surface first, not a marketing-only landing page. Include
working controls, dense but readable information, realistic states, strong
mobile behavior, and visible evidence that the app understands its domain.

Avoid auto-pushes, secret handling, destructive filesystem actions, and broad
rewrites that erase existing project state.
